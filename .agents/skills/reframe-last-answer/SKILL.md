# Reframe Last Answer

Recast **your own immediately-preceding answer** into a form the decision-maker can act on — without losing anything that changes a decision.

## The job in one line

The user is the carrier of intent; you are the analyst. A long, detailed answer is correct but hard to *act on*. This skill is the translation layer: **keep all the thinking, cut the reading load required to decide.**

## What to operate on

- The target is **your last assistant message** in this conversation. Reframe that, nothing else.
- If your last message was already short and decision-ready, say so in one line rather than padding it.
- If there is no prior substantive answer to reframe, ask which answer they mean. Don't invent one.
- Honor any steer in the arguments (audience, the single decision they care about, length).

## The non-negotiable rule

**Lossless on decisions. Lossy on process.**

- NEVER drop anything that would change the decision: the verdict, a caveat, a risk, a trade-off, a cost, a "this fix is only partial."
- DO drop the process: how you investigated, intermediate evidence, methodology, exhaustive enumeration.
- This is **reframing, not dumbing-down.** Dumbing-down destroys information. Reframing relocates it — detail moves to a pointer, it doesn't vanish.

## Principles (these are fixed; the shape is not)

1. **Verdict first.** Lead with the conclusion or recommendation, not the build-up. State the verdict plainly — never make the reader assemble it themselves.
2. **Plain language.** Layman terms. One analogy is allowed if it carries a hard concept. Strip jargon, or translate it inline.
3. **Make it decidable.** Surface the actual choice(s) and frame them so a decision is obvious — e.g. now vs later, this option vs that, the trade-off in each. Be honest about robustness ("solid", "partial", "band-aid").
4. **Offload, don't inline, the depth.** Point to where the full detail already lives (the report, the file, "ask me to expand"). Don't paste it back.
5. **Brief.** No wall of text. If it doesn't help a decision, it doesn't belong.

## Adaptive format (do NOT use a fixed template)

Pick the shape that fits the answer you're reframing. The principles above are constant; the layout flexes. Some shapes that tend to work — adapt freely, combine, or invent:

- **Decision / recommendation** → Problem (plain) · The call + how robust it is · What to decide (now / later). 
- **Comparison or trade-off** → a short table of options with a one-line verdict per option, then your pick.
- **Research / findings** → the 2-3 things that matter · what's still uncertain · what it implies. (No "fix to decide" here — don't force one.)
- **Risk / status** → bottom line · what could bite · what to do about it.

Choose by asking: *what does this person need to comprehend to act?* Build the answer backwards from that.

## Behavior

Produce the reframed answer directly — do not ask permission first, do not preface it with meta-commentary. The reframe **replaces** the wall of text; it does not annotate it.

## Self-check before sending

- Could the reader make the decision from this alone?
- Did I keep every decision-changing caveat?
- Did I cut process, not substance?
- Is it genuinely short?
